## ūüďĆ Overview

FraudSys is a full-stack insurance fraud detection and threat monitoring platform built for a cybersecurity portfolio project. It combines machine learning-based risk scoring with a real-time SOC (Security Operations Center) event log ‚ÄĒ simulating enterprise-grade tools like Wazuh, Splunk, and IBM QRadar.

Upload a CSV of insurance claims ‚Üí get instant risk classification, anomaly detection, confidence scores, and fraud flags ‚ÄĒ all visualized on a live dashboard.

## ‚ú® Features

### ūüĒź Authentication & Security
- **JWT-based auth** ‚ÄĒ signed tokens with 8-hour expiry, auto-logout on session end
- **2FA Email OTP** ‚ÄĒ 6-digit one-time password on every login (5-minute expiry)
- **Google OAuth** ‚ÄĒ single sign-on via Google
- **Brute force protection** ‚ÄĒ account locks after 5 failed attempts (15-minute lockout)
- **Input sanitization** ‚ÄĒ NoSQL injection prevention via `express-mongo-sanitize`, XSS stripping
- **Admin unlock** ‚ÄĒ admin can manually unlock locked accounts

### ūü§Ė ML Fraud Detection
- **Risk classification** ‚ÄĒ Low / Medium / High per claim
- **Confidence scoring** ‚ÄĒ 0‚Äď100% confidence per prediction
- **Anomaly detection** ‚ÄĒ flags statistically unusual claims
- **Rule-based flags** ‚ÄĒ `EXTREME_AMOUNT`, `EXCESSIVE_CLAIMS`, `HIGH_AMOUNT_HIGH_CLAIMS`, `ABNORMAL_CLAIM_RATIO`
- **Batch processing** ‚ÄĒ entire CSV processed in one pass (pure JS, no Python dependency)
- **Feature engineering** ‚ÄĒ 7 derived features from amount + claims

### ūüďä Dashboard
- **6 stat cards** ‚ÄĒ Total, Low, Medium, High risk, Anomalies, Avg Confidence
- **3 charts** ‚ÄĒ Claim Amount Timeline, Risk Distribution (pie), Confidence Score Distribution
- **Paginated case records** ‚ÄĒ 50 rows per page with risk badges, anomaly flags, confidence scores
- **Upload history** ‚ÄĒ clickable filenames load that upload's specific data on dashboard
- **Role-based views** ‚ÄĒ Admin gets full access; Analyst gets read-only dashboard

### ūüõ°ÔłŹ SOC Event Log (Admin Only)
- **Live security feed** ‚ÄĒ every auth event, upload, and anomaly logged automatically
- **Severity levels** ‚ÄĒ CRITICAL / HIGH / MEDIUM / LOW / INFO
- **Event categories** ‚ÄĒ AUTH / UPLOAD / ANOMALY / SYSTEM
- **Triage workflow** ‚ÄĒ change event status: OPEN ‚Üí INVESTIGATING ‚Üí RESOLVED
- **Stats cards** ‚ÄĒ Critical count, High count, Open events, Last 24h activity

## ūüöÄ Local Setup

### Prerequisites
- Node.js 20+
- MongoDB (local) or MongoDB Atlas URI
- Python 3.11+ with `pandas`, `scikit-learn`, `numpy`, `joblib` (optional ‚ÄĒ for local ML)

### Install & Run

```bash
# Clone the repo
git clone https://github.com/YOUR_USERNAME/insurance-fraud.git
cd insurance-fraud

# Install dependencies
npm install

# Set environment variables (create backend/.env)
MONGO_URI=mongodb://127.0.0.1:27017/fraudDB
JWT_SECRET=your-secret-key
GOOGLE_CLIENT_ID=your-google-client-id
EMAIL_USER=your-gmail@gmail.com      # optional ‚ÄĒ for real OTP emails
EMAIL_PASS=your-app-password         # optional

# Start MongoDB (Windows)
net start MongoDB

# Start server
node backend/server.js
```

Open `http://localhost:3000`
