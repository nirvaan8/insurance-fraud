## рҹ“Ң Overview

FraudSys is a full-stack insurance fraud detection and threat monitoring platform built for a cybersecurity portfolio project. It combines machine learning-based risk scoring with a real-time SOC (Security Operations Center) event log вҖ” simulating enterprise-grade tools like Wazuh, Splunk, and IBM QRadar.

Upload a CSV of insurance claims вҶ’ get instant risk classification, anomaly detection, confidence scores, and fraud flags вҖ” all visualized on a live dashboard.

---

## вңЁ Features

### рҹ”җ Authentication & Security
- **JWT-based auth** вҖ” signed tokens with 8-hour expiry, auto-logout on session end
- **2FA Email OTP** вҖ” 6-digit one-time password on every login (5-minute expiry)
- **Google OAuth** вҖ” single sign-on via Google
- **Brute force protection** вҖ” account locks after 5 failed attempts (15-minute lockout)
- **Input sanitization** вҖ” NoSQL injection prevention via `express-mongo-sanitize`, XSS stripping
- **Admin unlock** вҖ” admin can manually unlock locked accounts

### рҹӨ– ML Fraud Detection
- **Risk classification** вҖ” Low / Medium / High per claim
- **Confidence scoring** вҖ” 0вҖ“100% confidence per prediction
- **Anomaly detection** вҖ” flags statistically unusual claims
- **Rule-based flags** вҖ” `EXTREME_AMOUNT`, `EXCESSIVE_CLAIMS`, `HIGH_AMOUNT_HIGH_CLAIMS`, `ABNORMAL_CLAIM_RATIO`
- **Batch processing** вҖ” entire CSV processed in one pass (pure JS, no Python dependency)
- **Feature engineering** вҖ” 7 derived features from amount + claims

### рҹ“Ҡ Dashboard
- **6 stat cards** вҖ” Total, Low, Medium, High risk, Anomalies, Avg Confidence
- **3 charts** вҖ” Claim Amount Timeline, Risk Distribution (pie), Confidence Score Distribution
- **Paginated case records** вҖ” 50 rows per page with risk badges, anomaly flags, confidence scores
- **Upload history** вҖ” clickable filenames load that upload's specific data on dashboard
- **Role-based views** вҖ” Admin gets full access; Analyst gets read-only dashboard

### рҹӣЎпёҸ SOC Event Log (Admin Only)
- **Live security feed** вҖ” every auth event, upload, and anomaly logged automatically
- **Severity levels** вҖ” CRITICAL / HIGH / MEDIUM / LOW / INFO
- **Event categories** вҖ” AUTH / UPLOAD / ANOMALY / SYSTEM
- **Triage workflow** вҖ” change event status: OPEN вҶ’ INVESTIGATING вҶ’ RESOLVED
- **Stats cards** вҖ” Critical count, High count, Open events, Last 24h activity

## рҹҡҖ Local Setup

### Prerequisites
- Node.js 20+
- MongoDB (local) or MongoDB Atlas URI
- Python 3.11+ with `pandas`, `scikit-learn`, `numpy`, `joblib` (optional вҖ” for local ML)

### Install & Run

```bash
# Clone the repo
git clone https://github.com/YOUR_USERNAME/insurance-fraud.git
cd insurance-fraud

# Install dependencies
npm install

# Set environment variables (create backend/.env)
MONGO_URI=mongodb://127.0.0.1:27017/fraudDB
JWT_SECRET=your-secret-key
GOOGLE_CLIENT_ID=your-google-client-id
EMAIL_USER=your-gmail@gmail.com      # optional вҖ” for real OTP emails
EMAIL_PASS=your-app-password         # optional

# Start MongoDB (Windows)
net start MongoDB

# Start server
node backend/server.js
```

Open `http://localhost:3000`
